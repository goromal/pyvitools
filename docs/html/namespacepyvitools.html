Detailed Description

Select tools written in Python for processing and analyzing data for visual-inertial estimation applications.

Provides tools for:

  • Algebraic anipulation of vectors and SO(3)/SE(3) objects.
  • Differentiating and integrating the above.
  • Gaussian statistics analysis.
  • ROSbag processing.
  • Dataset manipulation.
  • (Coming soon) Plotting of all the above.

Calculates a list of indexed pose pairs \(((p_A, T_A), (p_B, T_B))\) for which robot \(A\) and robot \(B\) are viewing the same thing.

Parameters
poses_AList of (int, SE3) indexed poses for robot \(A\).
caminfo_ATuple of camera info parameters for robot \(A\): \((w,h,f_x,f_y)\) in pixels OR \((\text{FOV}_x, \text{FOV}_y)\) in radians.
max_depth_AMaximum depth at which robot \(A\) can be expected to resolve objects for data association.
poses_BList of (int, SE3) indexed poses for robot \(B\).
caminfo_BTuple of camera info parameters for robot \(B\): \((w,h,f_x,f_y)\) in pixels OR \((\text{FOV}_x, \text{FOV}_y)\) in radians.
max_depth_BMaximum depth at which robot \(B\) can be expected to resolve objects for data association.
min_intra_idx_diffMinimum index difference (for the same robot) between subsequent overlapping views. Helps remove dense clusters.
min_inter_idx_diffMinimum index difference (between the two robots) between any overlapping views. Useful if both robot paths are identical and belonging to the same robot.

Returns (and optionally visualizes) the zero-mean standard deviation of the error between noisy_data and true_data.

Assumes that noisy_data can be considered as true_data plus added Gaussian noise.

Parameters
error_dataNumpy array of length \(n\), equal to noisy data - true data.
covariance_gatingWhether or not to attempt to reject outliers via single-pass covariance gating.
title(Optional) Title for report and/or plot.
report_stats(Optional) Report diagnostics evaluating Gaussian assumption suitability.
plot(Optional) Visualize error probability distribution and Gaussian fit.

Instantiates a single-agent, visual-inertial monocular SLAM dataset from bagfile.

Parameters
bagfileThe file path of the input ROSbag file.
truth_topic(Optional) Topic to take truth values from. If not specified, will look for topics with message type Odometry, Pose, PoseStamped, PoseWithCovariance, PoseWithCovarianceStamped, Transform, TransformStamped, Point, or PointStamped.
image_topic(Optional) Topic to take images from. If not specified, will look for topics with message type Image.
imu_topic(Optional) Topic to take IMU measurements from. If not specified, will look for topics with message type Imu.
cam_info_topic(Optional) Topic to take camera info from. If not specified, will look for topics with message type CameraInfo.

Generates fake IMU measurements using the dataset's true state fields.

Parameters
gThe gravity vector, expressed in the world frame. A \(3\times 1\) numpy array.
sigma_etaA tuple of zero-mean noise standard deviations \((\sigma_{\eta,\text{accel}}, \sigma_{\eta,\text{gyro}})\).
kappaA tuple of bounds for the initial bias value \((\kappa_{\beta,\text{accel}}, \kappa_{\beta,\text{gyro}})\).
sigma_betaA tuple of random walk standard deviations \((\sigma_{\beta,\text{accel}}, \sigma_{\beta,\text{gyro}})\).

The default parameters generate noise- and bias-less measurements, assuming that the world z-axis points up. Implements the formulas derived at https://notes.andrewtorgesen.com/doku.php?id=public:imu#synthesizing_measurements.

Applies a rigid body transform to all IMU measurements, resulting in a transformed dataset.

Parameters
TThe transform SE(3) object, \(T_U^B\).

Assumes that you are trying to shift the IMU measurements from the vehicle body's center of mass ( \(B\)) to another frame still rigidly attached to the vehicle body ( \(U\)). Implements the formulas derived at https://notes.andrewtorgesen.com/doku.php?id=public:imu#shifting_measurements_in_se_3.
